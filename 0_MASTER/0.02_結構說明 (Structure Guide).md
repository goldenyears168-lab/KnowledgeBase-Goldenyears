資料夾結構說明 (Structure Guide)
--------------------------------------------------------------------------------
結構設計原則
1. 扁平化 (Flat Structure)
• 資料夾層級不超過 2 層
• 保持簡單清晰的目錄結構
• 避免過度嵌套
2. 清晰命名 (Clear Naming)
• 使用數字編號作為前綴，幫助自動排序
• 編號格式：主編號.子編號_類型_名稱
• 附錄文件：主編號.子編號_附錄_名稱
3. 代碼排序 (Code Sorting)
• 所有文件都有數字編號前綴
• 系統會按照數字順序自動排列
• 便於查找和維護

編號系統說明
主編號層級
• 0_ - 總綱文件
• 1_ - 客戶專案SOP
• 2_ - 夥伴角色規範
• 3_ - 總部支援SOP
子編號層級
• 1.01, 1.02 - 不同產品線
• 2.0, 2.1, 2.2 - 不同角色層級
• 3.01, 3.02, 3.03... - 不同支援功能
附錄編號
• 1.01.1, 1.01.2 - 1.01 的附錄
• 3.01.1, 3.02.1 - 3.01, 3.02 的附錄
--------------------------------------------------------------------------------
文件查找指南
尋找特定 SOP
1. 確定所屬類別（客戶專案 / 夥伴角色 / 總部支援）
2. 查看對應編號的資料夾
3. 按編號順序查找文件
尋找角色規範
1. 確定角色層級（總部 / 分店 / 外部）
2. 進入對應子資料夾
3. 按編號查找角色文件
尋找附錄文件
• 附錄文件編號與主文件相關
• 例如：1.01.1 是 1.01 的第一個附錄
• 通常放在同一資料夾中，便於查找
