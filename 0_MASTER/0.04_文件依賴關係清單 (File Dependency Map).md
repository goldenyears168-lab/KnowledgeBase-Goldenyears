4. 修改核心文件時的標準流程
4.1 修改前檢查
1. 確認修改範圍
    ◦ 確定要修改的文件
    ◦ 確定修改的內容（章節、流程、術語等）
2. 查找所有引用
    ◦ 使用本文件查找所有依賴文件
    ◦ 使用搜尋功能查找所有引用位置
3. 評估影響範圍
    ◦ 列出所有需要同步更新的文件
    ◦ 評估修改的複雜度
4.2 修改時注意事項
1. 保持引用格式一致
    ◦ 使用標準格式：📄 1.01_SOP工作室產品手冊.md
    ◦ 使用完整相對路徑，避免歧義
2. 版本管理
    ◦ 如果修改涉及版本號變更，需要更新所有引用
    ◦ 建議在文件名中保留版本號
3. 術語一致性
    ◦ 如果修改術語，需要同步更新所有文件
    ◦ 參考 0.01_0.01_好時有影總綱.md 中的「官方詞彙表」
4.3 修改後驗證
1. 檢查清單驗證
    ◦ 根據本文件的「修改檢查清單」逐項檢查
2. 搜尋驗證
    ◦ 使用搜尋功能確認所有引用已更新
    ◦ 確認沒有遺漏的引用
3. 文件完整性驗證
    ◦ 確認所有依賴文件仍然可以正常引用
    ◦ 確認文件路徑正確

5. 常見問題與解決方案
5.1 問題：修改文件名稱後，引用失效
解決方案：
1. 使用本文件查找所有依賴文件
2. 批量替換所有引用
3. 驗證所有引用已更新
5.2 問題：修改章節結構後，引用位置錯誤
解決方案：
1. 在修改前記錄原章節結構
2. 修改後更新所有章節引用
3. 使用本文件的檢查清單驗證
5.3 問題：術語變更後，多處引用不一致
解決方案：
1. 先在總綱文件的「官方詞彙表」中更新
2. 使用搜尋功能查找所有使用該術語的地方
3. 批量替換所有引用
