2.12_Spec_副攝 
1. 角色定位 
歡迎您，副攝 ！
您是「好時有影」組織的「SOP 執行者」與「第一線服務人員」。
您的核心使命，是透過**「100% 精準」地執行「📄 1.01_SOP工作室產品手冊.md」（以下簡稱「SOP_工作室」）中的「行政SOP」，為客戶、攝影師 、修圖師  創造一個「準確、順暢」**的服務體驗。
您是服務的「關鍵環節」，負責將「客戶的預約」精準轉換為「後台資訊系統 」中可被執行的「任務」，確保金流、資訊流、檔案流 100% 同步。
您是客戶服務品質與數據精準度的第一線執行者，您的回饋是我們改善SOP、讓組織更強大的關鍵。
--------------------------------------------------------------------------------
1.1 與攝影師的協作關係
您與 攝影師  是分工協作的關係：
• 您的職責： 行政SOP（接待、核對、結帳、系統更新、收店）
• 攝影師的職責： 拍攝服務SOP（拍攝、引導、挑圖）
• 協作方式： 透過標準化的「交接流程」確保資訊無縫傳遞
重要說明： 目前部分時段可能由攝影師同時擔任兩個角色（過渡期），但組織的目標是建立完全分工的協作模式。本文件定義的是您作為專職副攝的完整職責與標準。
--------------------------------------------------------------------------------
2. 核心職責 
您的職責是**「執行」，核心是「精準」**。
2.1 SOP 執行 
• 負責「📄 1.01_SOP工作室產品手冊.md」所有「行政SOP」的執行，包含：
    ◦ 「開店流程 」
    ◦ 「收店流程 」
2.2 客戶專案 - 接待 
• (關鍵職責) 執行客戶接待、引導，並**「核對 」**後台資訊系統上的預約資訊（包含急件、方案/身份/背景色、加購彩妝 等）。
• 您不需要推銷，您只需要核對。
• 完成接待後，將客戶轉交給 攝影師  或引導至自助妝髮區。
2.3 客戶專案 - 結帳 
• (關鍵職責) 您完全主導結帳流程。
• 接收 攝影師 的「口頭交接」，確認「最終選定張數」與「修圖備註」。
• 負責「收款SOP」，並立即在「後台資訊系統」中更新所有結帳欄位。
• 執行送客流程，體現作為職涯系統入口的祝福與陪伴。
2.4 系統完整性 
• (關鍵職責) 確保 後台資訊系統 的「即時性」與「準確性」。
• 負責從挑圖軟體（Lightroom）的備註欄位中**驗證/複製**攝影師的數位修圖備註，並將其精準登錄到後台資訊系統的「店內備註」欄位。若發現挑圖軟體中無備註記錄，應立即要求攝影師補充記錄。
• 負責執行「五星標記 」，確保標記張數與收款張數 100% 一致。
2.5 檔案與通知 
• 負責收店時的「批次上傳 」。
• 負責在 後台資訊系統 中更新 「批次上傳狀態」欄位，並通知修圖師
2.6 財務與總務 
• 負責「總務(總)」功能，維護店面環境整潔、資產點檢。
• 負責每日的「存錢與回報」SOP。
2.7 服務補救 
• 作為第一線窗口，執行「SOP_服務補救」手冊，安撫客戶並立即回報店長 。

3. 與攝影師的協作流程 
您與 攝影師  的協作，遵循以下標準流程：
3.1 階段 1：客戶接待（您主導）
您的職責：
1. 執行客戶接待與核對（見「2.2 客戶專案 - 接待」）
2. 完成接待後，通知 攝影師 準備接客戶
3. 若客戶有預約彩妝，引導至彩妝區；若無，引導至自助妝髮區
與攝影師的交接：
• 標準話術： 「[攝影師名字]，[客戶姓名] 已經完成接待，預約的是 [拍攝項目]，[若有急件] 有標記急件，麻煩您準備一下。」
• 或引導客戶至自助妝髮區後，通知攝影師：「[客戶姓名] 在自助妝髮區，好了會通知您。」
3.2 階段 2：攝影服務執行（攝影師主導，您支援）
您的職責：
• 維持環境整潔
• 準備下一位客戶的資料
• 若攝影師需要協助（例如：調整燈光、遞送道具），隨時支援
您不參與：
• 拍攝過程（除非攝影師明確請求協助）
• 挑圖引導（由攝影師獨立完成）
3.3 階段 3：協同結帳（您主導執行）
這是您與攝影師協作的關鍵環節，您完全主導結帳流程：
步驟 1：接收攝影師的交接（您接收）
• 攝影師陪同客戶走到櫃檯
• 攝影師向您口頭交接，標準交接話術 (必須包含三要素)：
    ◦ 「[您的名字]，這位是 [客戶姓名] 先生/小姐，最終選定了 [數量] 張照片。修圖備註是：『[清晰簡述修圖需求，例如：右臉痘痘、膚色調亮、背景微調]』。」
• 攝影師將「記憶卡」交給您
• (標準流程) 您應同時從挑圖軟體（Lightroom）中**驗證/複製**修圖備註。若挑圖軟體中無備註記錄，應立即要求攝影師補充記錄。
• (例外情況) 在以下特殊情況下，允許使用「轉錄」方式（先記錄口頭交接的備註，事後再驗證）：
    ◦ 挑圖軟體無法開啟或系統異常
    ◦ 攝影師已離場且無法立即補充記錄
    ◦ 緊急情況（例如：客戶有急事需立即離開）
    ◦ **重要：** 使用轉錄時，必須在24小時內完成事後驗證，並在「店內備註」中標註「[轉錄，已於YYYY-MM-DD驗證]」
步驟 2：您確認理解（您執行）
• 您複述關鍵資訊以確認理解
• 標準確認話術： 「好的，收到。所以是 [數量] 張，修圖備註『[重複備註]』，對嗎？」
• 若資訊不清楚，立即詢問攝影師澄清
• 確認無誤後，告知攝影師：「好的，確認無誤，您可以準備下一位客戶了。」
步驟 3：交接確認期
• 目的： 確保資訊準確，降低後續修正成本
• 流程：
    1. 您立即執行「五星標記」，標記攝影師交接的張數
    2. 您與客戶口頭確認標記的照片是否正確
    3. 標準確認話術： 「好的，這 [數量] 張幫您標記好了，就是這幾張沒錯齁？」
    4. 若客戶確認無誤，告知攝影師：「[攝影師名字]，[客戶姓名] 的照片已經確認無誤，您可以準備下一位客戶了。」
    5. 若發現問題（張數不符、照片不對），立即請攝影師回來確認
• 時間： 約 2-3 分鐘
• 重要性： 這是降低錯誤率的關鍵機制，務必執行
步驟 4：您執行收款與系統更新（您主導）
• 您執行收款SOP（見「4.2 關鍵SOP：收款」）
• 您同時完成「後台資訊系統更新」（見「4.3 關鍵SOP：標記與更新」）
步驟 5：您執行送客（您主導）
• 您執行送客流程，體現「祝福儀式」（見「4.4 送客流程」）
3.4 階段 4：收店（您主導，攝影師協助）
您的職責：
• 執行收店流程、批次上傳、通知修圖師
• 確認所有系統記錄完整
攝影師的協助：
• 器材歸位
• 環境整理（棚內）

4. 異常情況處理SOP 
4.1 情況A：攝影師交接資訊不完整
處理流程：
1. 立即詢問： 「不好意思，[攝影師名字]，您剛才說 [重複不清楚的部分]，可以再確認一下嗎？」
2. 若涉及修圖備註： 務必確認到具體、可執行的程度（避免「修好看一點」這種模糊描述）
3. 若挑圖軟體中無修圖備註記錄： 立即要求攝影師在挑圖軟體中補充記錄，確認無誤後再進行系統登錄
4. 確認後： 再次複述給攝影師聽，確保理解正確
5. 若攝影師已離場： 立即聯繫 店長 ，記錄於「後台資訊系統」的「店內備註」，標註「待確認」
4.2 情況B：記憶卡標記與交接張數不符
處理流程：
1. 先暫停系統登錄
2. 立即檢查： 查看記憶卡實際標記的張數
3. 以實際標記為準： 以實際標記的張數為準，而非攝影師口頭交接的張數
4. 記錄差異： 記錄於「後台資訊系統」的「店內備註」，標註「張數差異：[交接張數] vs [實際標記張數]」
5. 向店長反映： 立即向 店長  反映，評估是否需要補拍或調整
4.3 情況C：攝影師忘記交接
處理流程：
1. 主動詢問： 若攝影師陪同客戶走到櫃檯但未主動交接，您應主動詢問：「[攝影師名字]，[客戶姓名] 選了幾張？有什麼修圖備註嗎？」
2. 若攝影師已離場： 立即聯繫 店長 
3. 記錄待確認： 記錄於「後台資訊系統」的「店內備註」，標註「待確認」
4. 向店長反映： 向 店長 反映，避免類似情況再次發生
4.4 情況D：客戶在結帳時改變主意
處理流程：
1. 立即更新： 以客戶最終確認的張數為準
2. 通知攝影師： 若攝影師仍在現場，立即通知攝影師更正
3. 更新系統： 更新「後台資訊系統」的相關欄位
4. 記錄變更： 記錄於「店內備註」，標註「結帳時變更：[原張數] → [新張數]」
4.5 情況E：挑圖軟體中無修圖備註記錄
處理流程：
1. 立即要求攝影師在挑圖軟體中補充記錄修圖備註
2. 確認記錄無誤後，再進行系統登錄
3. 若攝影師已離場： 記錄於「後台資訊系統」的「店內備註」，標註「待確認：挑圖軟體中無備註記錄」
4. 向店長反映： 立即向 店長  反映，避免類似情況再次發生
4.6 情況F：系統異常時的彈性處理
當遇到以下系統異常情況時，允許使用彈性處理方式：
• **挑圖軟體無法開啟或當機：**
    ◦ 處理方式： 先使用「轉錄」方式，記錄攝影師口頭交接的修圖備註
    ◦ 事後驗證： 必須在系統恢復後24小時內，重新開啟挑圖軟體驗證備註
    ◦ 記錄方式： 在「店內備註」中標註「[轉錄，系統異常，已於YYYY-MM-DD驗證]」
• **後台資訊系統無法登錄：**
    ◦ 處理方式： 先使用替代方式記錄（紙本或LINE通知店長），完成收款和標記
    ◦ 事後補登： 必須在系統恢復後24小時內，將所有資訊補登到後台資訊系統
    ◦ 驗證機制： 補登後需請店長或攝影師協助驗證資訊正確性
    ◦ 記錄方式： 在「店內備註」中標註「[補登，系統異常，已於YYYY-MM-DD補登並驗證]」
• **網路連線異常：**
    ◦ 處理方式： 先完成現場作業（收款、標記），使用離線方式記錄
    ◦ 事後同步： 網路恢復後立即同步到後台資訊系統
    ◦ 驗證機制： 同步後需確認所有資訊完整無誤
• **重要原則：**
    ◦ 系統異常不應影響客戶服務品質，優先完成現場作業
    ◦ 所有彈性處理都必須有「事後驗證」機制，確保資訊最終準確
    ◦ 必須在24小時內完成補登和驗證，並向店長回報
    ◦ 若無法在24小時內完成，必須立即向店長反映，尋求協助

6. 工作重點 
我們透過「系統」、「SOP」與「客戶回饋」共同檢視您的貢獻：
1. 系統準確度 
    ◦ 持續提升後台資訊系統的登錄準確性，確保店內備註完整、急件標記準確、五星標記的檔案與收款張數一致。
    ◦ 透過系統數據持續檢視並改善，逐步提升執行品質，最終達到穩定且高品質的執行能力。
    ◦ **注意：** 具體的準確度標準請參考「6.4 情境化績效標準」，會根據您的發展階段和特殊情況進行調整。
2. SOP 執行率 
    ◦ 確實執行「📄 1.01_SOP工作室產品手冊.md」及「📄 3.06_SOP_總務 .md」的所有流程。
    ◦ 確保開店SOP、批次上傳與LINE通知等關鍵流程依SOP完成，在特殊情況下能妥善處理例外狀況。
    ◦ **注意：** 具體的執行率標準請參考「6.4 情境化績效標準」，會根據您的發展階段和特殊情況進行調整。
3. 客戶滿意度 
    ◦ 持續關注客戶回饋（Google評論、後台資訊系統回饋欄位）中關於接待服務、結帳流程、整體體驗的評價。
    ◦ 確保接待親切專業、結帳流程順暢、客戶感到被尊重，在追求系統準確度的同時，維護良好的客戶體驗。
6.4 績效標準 
• **基本目標：**
    ◦ 系統準確度： ≥ 99%（認證夥伴標準）
    ◦ SOP 執行率： = 100%（所有流程依SOP完成）
• **原則：** 我們追求「持續改善」。標準會根據您的成長階段和實際情況動態調整，初期允許較高的學習容錯空間，逐步提升至認證標準。

2. 若攝影師已離場，立即聯繫 店長 
3. 記錄於「後台資訊系統」的「店內備註」，標註「待確認」
4. 向 店長 反映，避免類似情況再次發生
Q2：如果攝影師交接的資訊不清楚怎麼辦？
A： 立即澄清，不要猜測：
1. 標準澄清話術： 「不好意思，[攝影師名字]，您剛才說 [重複不清楚的部分]，可以再確認一下嗎？」
2. 若涉及「修圖備註」，務必確認到具體、可執行的程度
3. 確認後，再次複述給攝影師聽
Q3：如果系統登錄時發現攝影師交接的張數與實際標記不符怎麼辦？
A： 立即與攝影師確認：
1. 先暫停系統登錄
2. 立即聯繫攝影師（若在現場）或查看記憶卡實際標記
3. 以實際標記的張數為準
4. 記錄差異，向 店長 反映
Q4：我如何確保與攝影師的協作順暢？
A： 建立「主動溝通」的習慣：主動確認、主動詢問、主動回報。與常合作的攝影師建立工作默契，但仍需遵循標準流程。
Q5：交接確認期一定要執行嗎？
A： 是的，這是必須執行的關鍵機制。約 2-3 分鐘，但可以避免後續修正的更大成本，是降低錯誤率最有效的方式。
