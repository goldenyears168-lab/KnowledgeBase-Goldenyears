3.11_SOP_審核流程 (Review Process)
1. SOP 總覽 (Overview)
1.1 手冊目的與核心原則
• 目的 (Purpose): 本手冊旨在建立一套標準化的審核工作流程，讓店長能夠高效地審核自動化夥伴的執行結果，確保品質與準確性。
• 核心原則 (Core Principles):
    1. 快速決策：對於標準化的事項，快速審核並批准
    2. 異常關注：對於異常情況，深入分析並決策
    3. 反饋機制：審核後提供反饋給自動化夥伴，持續改善流程
    4. 品質把關：確保所有自動化執行結果符合品牌標準與SOP要求
1.2 適用範圍
• 本SOP適用於店長 (Hub Manager) 審核自動化夥伴的執行結果
• 涵蓋所有自動化執行的工作：營運報表、行銷內容、財務報表、人才資料等
1.3 權責單位
• SOP 擁有者 (Owner): 執行長 (CEO)
• SOP 執行者 (Executor): 店長 (Hub Manager)
2. 審核工作分類 (Review Work Categories)
2.1 每日審核 (Daily Review)
時間點： 每日早上開始工作前（約 **10-15 分鐘**）
**審核重點調整：** 每日審核應聚焦於 P0/P1 緊急異常，詳細數據分析改為每週進行，釋放店長時間專注於現場管理與品質控管。
審核項目：
1. **異常事件清單（優先處理 P0/P1 級別）**
    ◦ 系統異常事件（僅限 P0/P1：系統錯誤、急件未完成等影響營運的異常）
    ◦ 營運異常事件（僅限 P0/P1：急件達成率異常下降、系統錯誤率異常上升等）
    ◦ 需要關注的客戶投訴（僅限 P0/P1：緊急客訴、重大服務問題）
2. **需要決策的事項（僅限緊急事項）**
    ◦ 自動化夥伴標記的需要人工判斷的訊息（P0/P1 優先級）
    ◦ 需要店長立即決策的異常情況
3. **關鍵指標快速掃描（非詳細分析）**
    ◦ 僅快速掃描關鍵指標（例如：急件達成率、系統錯誤率），確認是否在正常範圍內
    ◦ **注意：** 詳細的營運數據分析（預約數、完成數、客戶滿意度數據等）請參考「2.2 每週審核」
審核標準：
• 異常是否已標記並說明
• 是否需要立即處理（P0 緊急 / P1 重要）
• 關鍵指標是否在正常範圍內（快速判斷，無需深入分析）
**審核原則：**
• 快速決策：對於標準化的事項，快速審核並批准
• 異常優先：僅處理需要立即關注的 P0/P1 異常，其他事項可延至每週審核
• 時間控制：每日審核時間控制在 10-15 分鐘內，避免過度深入分析
2.2 每週審核 (Weekly Review)
時間點： 每週一早上（約 30 分鐘）
**審核重點調整：** 每週審核包含詳細的營運、行銷、財務報表分析。
**抽樣審核機制：** 為提升審核效率，非關鍵項目採用抽樣審核方式，僅詳細審核關鍵指標和異常項目。
審核項目：
1. **週營運摘要（詳細分析）**
    ◦ **前一日營運報表詳細數據**（從每日審核移至本節）：
        ▪ 預約數、完成數、急件數、錯誤數
        ▪ 系統錯誤率、急件達成率（詳細趨勢分析）
        ▪ 客戶滿意度數據（與歷史數據對比）
    ◦ 關鍵指標趨勢（與上週對比）
    ◦ 異常事件清單（彙整整週的異常事件，分析異常模式）
    ◦ 營運效率分析
2. 行銷成效報告（抽樣審核）
    ◦ **重點審核：** 僅審核關鍵指標（客戶獲取成本 CAC、轉換率）
    ◦ **抽樣審核：** 社群媒體觸及數、互動率採用抽樣方式（例如：僅審核前3個主要平台，或僅審核異常下降的平台）
    ◦ **原則：** 若關鍵指標正常，其他指標可快速掃描；若關鍵指標異常，則深入分析相關指標
3. 財務報表（重點審核）
    ◦ **重點審核：** 預算執行狀況、分潤計算結果（關鍵財務指標）
    ◦ **抽樣審核：** 週收支表的詳細項目採用抽樣方式（例如：僅審核大額支出、異常項目）
    ◦ **原則：** 若預算執行正常且分潤計算無誤，詳細收支項目可快速掃描
4. 夥伴工作狀況（重點審核）
    ◦ **重點審核：** 需要關注的工作異常、關鍵夥伴的工作品質
    ◦ **抽樣審核：** 一般夥伴工作數據採用抽樣方式（例如：僅審核新進夥伴、績效異常的夥伴）
    ◦ **原則：** 若無異常，可快速掃描；若有異常，則深入分析
審核標準：
• 指標趨勢是否正常
• 是否有異常需要關注
• 是否需要調整策略
2.3 每月審核 (Monthly Review)
時間點： 每月初（約 1 小時）
**重點審核機制：** 每月審核採用重點審核模式，僅詳細審核關鍵指標和異常項目，其他項目採用快速掃描或抽樣方式。
審核項目：
1. 月營運數據報告（重點審核）
    ◦ **重點審核：** 月營收、客戶增長、關鍵指標異常項目
    ◦ **快速掃描：** 夥伴績效、SOP執行率（若無異常，快速掃描即可）
    ◦ **原則：** 若關鍵指標正常，其他指標可快速掃描；若關鍵指標異常，則深入分析相關指標
2. 夥伴績效報告（重點審核）
    ◦ **重點審核：** 需要關注的夥伴問題、新進夥伴的培訓進度
    ◦ **快速掃描：** 一般夥伴的績效評估結果（若無異常，快速掃描即可）
    ◦ **抽樣審核：** 培訓進度追蹤採用抽樣方式（例如：僅審核新進夥伴、需要關注的夥伴）
    ◦ **原則：** 若無異常，可快速掃描；若有異常，則深入分析
3. 預算執行報告（重點審核）
    ◦ **重點審核：** 預算超支情況、預算調整建議
    ◦ **快速掃描：** 預算執行狀況（若無超支，快速掃描即可）
    ◦ **原則：** 若預算執行正常，可快速掃描；若有超支，則深入分析
4. 修圖師作品審查（品質抽檢 - 重點審核）
    ◦ (關鍵職責) 每月定期審查修圖師 (Verified Retoucher) 當月完成的作品
    ◦ **重點審核：** 僅審核需要關注的修圖師作品（例如：新進修圖師、收到客戶回饋的修圖師、上個月有品質問題的修圖師）
    ◦ **抽樣審核：** 其他修圖師作品採用抽樣方式（例如：每位修圖師抽檢 2-3 件作品）
    ◦ 確保品質符合品牌風格與客戶需求
    ◦ 識別需要改進的地方，提供培訓建議
    ◦ 優秀作品可作為範例分享，持續追蹤改善成效
    ◦ 參考文件： 《3.10_SOP_品質管理 (Quality Management).md》第 5 節
5. 月度檢討與規劃（重點審核）
    ◦ **重點審核：** 僅檢討需要調整的策略和需要關注的問題
    ◦ **快速掃描：** 其他項目快速掃描即可
    ◦ 根據審核結果進行月度檢討
    ◦ 制定下月工作重點
    ◦ 提出改善建議
審核標準：
• 月度目標是否達成
• 是否有需要調整的策略
• 是否有需要關注的問題
• 修圖師作品是否符合品牌風格與品質標準
2.4 即時審核 (Real-time Review)
時間點： 收到通知後立即處理
審核項目：
1. 系統異常通知
    ◦ 系統錯誤、急件未完成等異常
    ◦ 需要立即處理的系統問題
2. 客戶投訴
    ◦ 需要人工判斷的客戶投訴
    ◦ 需要店長決策的服務補救
3. 需要人工判斷的訊息
    ◦ 自動化夥伴標記的社群媒體訊息
    ◦ 需要店長決策的客戶回覆
審核標準：
• 是否需要立即處理
• 處理方式是否符合SOP
• 是否需要升級處理
3. 審核標準與檢查清單 (Review Standards & Checklists)
3.1 營運報表審核標準
檢查清單：
• [ ] 數據是否完整（所有必要欄位都有數據）
• [ ] 數據是否合理（與歷史數據對比，無明顯異常）
• [ ] 異常是否已標記並說明
• [ ] 關鍵指標是否達標
• [ ] 是否需要立即處理的問題
審核決策：
• 通過：數據完整、合理，無異常需要處理
• 需要調整：數據有誤或需要補充
• 需要處理：發現異常需要立即處理
3.2 行銷內容審核標準
檢查清單：
• [ ] 內容是否符合品牌規範（語氣、風格、視覺）
• [ ] 內容是否符合內容策略
• [ ] 文案是否有錯誤（錯字、語法）
• [ ] 視覺是否符合品牌標準
• [ ] 排程時間是否合適
審核決策：
• 通過：內容符合品牌規範，可以發布
• 需要修改：內容需要調整後再發布
• 拒絕：內容不符合品牌規範，需要重新產製
3.3 財務報表審核標準
檢查清單：
• [ ] 數據是否完整（收入、支出、分潤）
• [ ] 計算是否正確（分潤計算、預算執行率）
• [ ] 是否有異常（預算超支、分潤錯誤）
• [ ] 是否符合財務SOP
審核決策：
• 通過：數據完整、計算正確，可以發放
• 需要調整：數據有誤或需要補充
• 需要處理：發現異常需要立即處理
3.4 人才資料審核標準
檢查清單：
• [ ] 候選人資料是否完整
• [ ] 是否符合角色說明書要求
• [ ] 篩選標準是否正確
• [ ] 是否需要進一步評估
審核決策：
• 通過：符合要求，可以進入面試
• 需要補充：資料不完整，需要補充
• 拒絕：不符合要求，不進入面試
3.5 修圖師作品審查標準
檢查清單：
• [ ] 品牌風格一致性（是否符合「好時有影」品牌色調標準）
• [ ] 修圖品質是否符合標準（亮度、對比、膚質處理等）
• [ ] 客戶需求是否準確執行（是否依照店內備註進行修圖）
• [ ] 是否有需要改進的地方（技術問題、風格偏差）
• [ ] 優秀作品是否已標記作為範例
審核重點：
• 品牌風格一致性： 確保所有作品符合「好時有影」的品牌美學標準
• 品質標準： 檢查照片是否明亮通透、細節清晰、膚質處理自然
• 客戶需求執行： 確認修圖師是否依照後台資訊系統中的店內備註進行修圖
• 改進追蹤： 識別需要培訓或改進的修圖師，提供具體建議並追蹤改善成效
審核決策：
• 通過：作品符合品牌標準與品質要求
• 需要改進：發現品質或風格問題，需與修圖師溝通並提供培訓建議
• 需要重修：作品不符合標準，需要求修圖師重新處理
處理方式：
• 發現問題時，與修圖師進行一對一溝通與培訓
• 優秀作品可作為範例分享給其他修圖師
• 持續追蹤改善成效，確保品質持續提升
4. 審核決策流程 (Review Decision Process)
4.1 標準審核流程
自動化夥伴執行 → 生成結果 → 發送給店長 → 店長審核 → 決策
                                                      │
                                                      ├─ 通過 → 執行
                                                      ├─ 需要調整 → 回饋給自動化夥伴 → 調整後重新審核
                                                      └─ 拒絕 → 回饋給自動化夥伴 → 重新執行
4.2 審核決策類型
1. 通過 (Approve)
    ◦ 結果符合標準，可以執行
    ◦ 無需調整，直接批准
2. 需要調整 (Need Adjustment)
    ◦ 結果基本符合標準，但需要小幅調整
    ◦ 提供具體調整建議
    ◦ 調整後重新審核
3. 拒絕 (Reject)
    ◦ 結果不符合標準，需要重新執行
    ◦ 提供拒絕原因與改進建議
    ◦ 重新執行後再次審核
4.3 審核反饋機制
反饋內容：
• 審核結果（通過/需要調整/拒絕）
• 具體問題說明
• 改進建議
• 優先級（緊急/重要/一般）
反饋方式：
• Email 回覆
• 後台系統標記
• 即時通知（緊急情況）
5. 異常處理流程 (Exception Handling Process)
5.1 異常識別
異常類型：
• 數據異常（數據不合理、缺失）
• 系統異常（系統錯誤、當機）
• 內容異常（不符合品牌規範）
• 流程異常（不符合SOP）
5.2 異常處理流程
異常識別 → 標記異常 → 通知店長 → 店長審核 → 決定處理方式 → 執行處理
處理方式：
• 立即處理：緊急異常，需要立即處理
• 計劃處理：非緊急異常，可以計劃處理
• 忽略：非異常，可以忽略
5.3 異常升級機制
升級條件：
• 影響客戶體驗的異常
• 影響財務的異常
• 需要執行長決策的異常
升級流程：
• 店長識別需要升級的異常
• 通知執行長
• 執行長決策後執行
6. 審核效率提升策略 (Review Efficiency Strategies)
6.1 優先級管理
優先級分類：
• P0（緊急）：立即處理，影響客戶體驗或財務
• P1（重要）：當日處理，影響營運效率
• P2（一般）：本週處理，不影響營運
• P3（低優先級）：有時間再處理
6.2 批量審核
適用場景：
• 每日審核：批量審核前一日所有報表
• 每週審核：批量審核上週所有報告
• 每月審核：批量審核上月所有報告
效率提升：
• 建立審核模板
• 使用審核檢查清單
• 快速決策標準化事項
6.3 自動化輔助
自動化工具：
• 自動標記異常
• 自動生成審核摘要
• 自動提醒審核時間
6.4 審核自動化機會識別 (Review Automation Opportunities)
• **目的：** 識別可以透過自動化減少人工審核負擔的項目，提升審核效率。
• **自動化機會分類：**
    1. **完全自動化：** 可以完全由系統自動審核，無需人工介入
        ◦ 範例： 數據格式檢查、必填欄位檢查、數值範圍檢查
        ◦ 實施方式： 透過自動化夥伴建立自動驗證規則
    2. **半自動化：** 系統自動標記異常，人工僅需審核異常項目
        ◦ 範例： 異常數據標記、趨勢異常檢測、預算超支提醒
        ◦ 實施方式： 透過自動化夥伴建立異常檢測機制
    3. **自動化摘要：** 系統自動生成審核摘要，人工僅需審核摘要
        ◦ 範例： 自動生成週報摘要、月度關鍵指標摘要
        ◦ 實施方式： 透過自動化夥伴建立報表摘要生成機制
• **優先實施項目：**
    ◦ P0（立即實施）： 異常數據自動標記、預算超支自動提醒
    ◦ P1（本季實施）： 自動生成審核摘要、關鍵指標自動對比
    ◦ P2（本年度實施）： 完全自動化的數據驗證、智能異常檢測
• **效益評估：**
    ◦ 目標： 透過自動化減少 30-40% 的審核時間
    ◦ 追蹤方式： 每月統計審核時間變化，評估自動化效益
7. 審核品質保證 (Review Quality Assurance)
7.1 審核準確性
確保方法：
• 使用審核檢查清單
• 對比歷史數據
• 交叉驗證關鍵數據
7.2 審核一致性
確保方法：
• 建立標準審核流程
• 使用統一審核標準
• 定期檢討審核品質
7.3 持續改進
改進機制：
• 收集審核反饋
• 分析審核效率
• 改善審核流程

每日審核檢查清單：

• [ ] 異常事件已處理

每週審核檢查清單：
